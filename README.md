# Cronograma de Estudos — Do Zero até IA 🚀

> Objetivo: sair do zero absoluto em programação até ter uma base sólida em Python e Inteligência Artificial, 100% gratuito, para depois entrar em uma faculdade/curso com vantagem.

**Regra de ouro:** consistência > intensidade. Melhor 1h todo dia do que 6h só no fim de semana. Marque os checkboxes `[ ]` → `[x]` conforme for completando.

---

## FASE 1 — Fundamentos de Programação (Semanas 1 a 6)

Objetivo: ficar confortável com lógica de programação e Python básico.

### Semana 1 — Primeiros passos
- [ ] Assistir aulas 1 a 5 do **Curso em Vídeo - Lógica de Programação** (YouTube, Gustavo Guanabara)
- [ ] Criar conta no **Google Colab** (colab.research.google.com)
- [ ] Escrever seu primeiro "Olá, mundo!" em Python no Colab
- [ ] Entender: o que é uma variável, o que é um algoritmo

### Semana 2 — Lógica básica
- [ ] Continuar Curso em Vídeo (variáveis, tipos de dados, operadores)
- [ ] Praticar exercícios simples (calculadora, conversor de temperatura)
- [ ] Criar conta no **GitHub** (vai guardar seus códigos lá desde já)

### Semana 3 — Estruturas de decisão e repetição
- [ ] Aprender `if / else` (condicionais)
- [ ] Aprender `for` e `while` (loops)
- [ ] Fazer 5 exercícios pequenos (ex: par ou ímpar, tabuada, jogo de adivinhação)

### Semana 4 — Estruturas de dados
- [ ] Aprender listas e dicionários em Python
- [ ] Praticar manipulação de listas (adicionar, remover, percorrer)
- [ ] Mini projeto: lista de tarefas simples no terminal

### Semana 5 — Funções
- [ ] Aprender a criar e usar funções
- [ ] Entender parâmetros e retorno de valores
- [ ] Refazer os exercícios anteriores usando funções

### Semana 6 — Consolidação
- [ ] Fazer um mini projeto final: pode ser uma calculadora completa, um jogo da forca em texto, ou similar
- [ ] Subir o projeto no seu GitHub
- [ ] Revisar tudo que foi visto até aqui

---

## FASE 2 — Matemática Essencial (rodar em paralelo, Semanas 3 a 8)

Não precisa parar a Fase 1 pra isso — pode intercalar 2-3x por semana.

- [ ] **Khan Academy (PT-BR)**: módulo de Álgebra básica
- [ ] **Khan Academy**: módulo de Estatística e Probabilidade
- [ ] Entender: média, mediana, desvio padrão, o que é uma probabilidade
- [ ] Entender: o que é um vetor e uma matriz (noção básica, sem precisar fazer contas complexas)

---

## FASE 3 — Introdução à IA e Machine Learning (Semanas 7 a 14)

### Semana 7 — Conceitos gerais
- [ ] Fazer o curso **Elements of AI** (elementsofai.com) — introdução sem precisar programar
- [ ] Entender as diferenças: IA, Machine Learning, Deep Learning

### Semana 8 — Google AI Essentials
- [ ] Completar o curso **Google AI Essentials** (gratuito)
- [ ] Anotar os principais conceitos e termos novos (crie um glossário pessoal)

### Semanas 9 a 10 — Python para dados
- [ ] **Kaggle Learn**: curso "Python"
- [ ] **Kaggle Learn**: curso "Pandas" (manipulação de dados)
- [ ] Praticar com um dataset simples (ex: dados de filmes, jogos, etc.)

### Semanas 11 a 12 — Primeiros passos em Machine Learning
- [ ] **Kaggle Learn**: curso "Intro to Machine Learning"
- [ ] **Kaggle Learn**: curso "Intermediate Machine Learning"
- [ ] Entender: o que é treinar um modelo, o que são dados de treino/teste

### Semanas 13 a 14 — Prática real
- [ ] Participar de uma competição iniciante no Kaggle (ex: "Titanic - Machine Learning from Disaster", clássica pra iniciantes)
- [ ] Documentar o processo no GitHub
- [ ] Escrever um resumo do que aprendeu (isso ajuda a fixar o conteúdo)

---

## FASE 4 — Aprofundamento e Direção (Semana 15 em diante)

Aqui você já vai ter noção do que mais gosta. Escolha um caminho:

- [ ] Se curtiu programar modelos → pesquisar cursos gratuitos de **Deep Learning** (ex: fast.ai)
- [ ] Se curtiu análise de dados → aprofundar em **Ciência de Dados** e **SQL**
- [ ] Se curtiu usar ferramentas de IA no dia a dia → explorar **prompt engineering** e automações
- [ ] Começar a pesquisar cursos técnicos, bolsas (Prouni, Fies) ou cursos gratuitos de prefeituras/SENAI voltados pra IA

## Dicas para não desistir

1. **Estude um pouco todo dia**, mesmo que só 30 minutos — é melhor que maratonar 1x por semana.
2. **Não pule etapas.** A tentação de ir direto pra IA sem base de programação é grande, mas vai te atrasar.
3. **Erre bastante.** Código quebrado é parte do processo, não é sinal de que você não é capaz.
4. **Mostre o que faz.** Poste seus pequenos projetos no GitHub — isso vira portfólio pra faculdade ou emprego futuro.
5. Volte aqui sempre que quiser ajustar o ritmo, tirar dúvida de algum conceito, ou revisar algo que travou.
